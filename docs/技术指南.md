# Library-System-JavaFX 图书管理系统技术指南

## 1. 项目概述

### 1.1 项目定位

本项目是一个基于 **Spring Boot** 和 **JavaFX** 构建的单体图书管理系统，提供完整的用户注册、登录、认证、授权、图书管理和借阅管理功能。服务端逻辑和客户端 GUI 集成在同一模块中，无需单独部署服务端。

### 1.2 技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| Java | 17 | 编程语言 |
| Spring Boot | 3.2.0 | 应用框架 |
| Spring Security | 6.x | 安全框架 |
| Spring Data JPA | 3.2.x | 数据访问 |
| JWT (JJWT) | 0.12.3 | 令牌认证 |
| MySQL | 8.x | 数据库 |
| JavaFX | 21.0.2 | 客户端 GUI |
| Lombok | 1.18.30 | POJO 简化 |

## 3. 核心流程详解

### 3.1 用户注册流程

```
RegisterView → AuthenticationService.register()
                 ├─ 接收 RegisterRequest (username, password, phone)
                 ├─ 参数校验 (@NotBlank, @Size)
                 ├─ 检查用户名是否已存在
                 ├─ 检查手机号是否已存在
                 ├─ BCrypt 加密密码
                 ├─ 保存用户到数据库（默认角色 USER）
                 └─ 返回 UserDTO
```

### 3.2 用户登录流程

```
LoginView → AuthenticationService.login()
              ├─ 接收 LoginRequest (account, password)
              ├─ 参数校验
              ├─ 根据 account 查询用户（仅支持 username）
              ├─ BCrypt 密码匹配校验
              ├─ JWT 生成令牌（含 userId、username）
              ├─ TokenStore 存储 Token
              └─ 返回 LoginResponse (id, username, email, phone, token, role)
```

### 3.3 JWT 认证流程

```
Service 方法调用 → SecurityConfig 配置的过滤器链
                    ├─ 从 TokenStore 获取 Token
                    ├─ 验证 Token 有效性
                    ├─ 解析 Token 获取 username
                    ├─ 加载 UserDetails（含角色权限）
                    └─ 设置 SecurityContext 认证信息
```

### 3.4 权限控制流程

```
Service 方法标注 @PreAuthorize
    ├─ hasRole('ADMIN')         → 仅管理员可访问
    ├─ isAuthenticated()        → 登录用户可访问
    └─ @userServiceImpl.isOwnUser(#id)  → 仅本人可访问
```

### 3.5 图书借阅流程

```
BookListTab → BookService.borrowBook(userId, request)
                ├─ 接收 userId 和 BorrowRequest (bookId, borrowDays)
                ├─ 根据 userId 查询用户信息
                ├─ 检查图书是否存在
                ├─ 检查图书库存是否充足
                ├─ 检查用户是否已借阅该图书
                ├─ 扣减图书库存
                ├─ 创建借阅记录（设置借阅时间、应还时间）
                └─ 返回 BorrowRecordDTO
```

> **安全设计**：`borrowBook` 方法直接接受 `userId` 参数，避免在多线程环境下 `SecurityContextHolder` 获取不到认证信息的问题。

### 3.6 图书归还流程

```
MyBorrowTab → BookService.returnBook(recordId)
                ├─ 根据 recordId 查询借阅记录
                ├─ 检查记录是否处于借阅状态
                ├─ 查询图书信息
                ├─ 增加图书库存
                ├─ 更新借阅记录（设置归还时间、状态改为已归还）
                └─ 返回 BorrowRecordDTO
```

## 5. 安全机制

### 5.1 Spring Security 配置

```java
// 核心配置要点
- CSRF: 禁用（桌面应用场景）
- Session: 无状态（STATELESS）
- 白名单: /api/**（单体应用内部调用）
- 过滤器链: JwtAuthenticationFilter → UsernamePasswordAuthenticationFilter
```

### 5.2 JWT 令牌

**Token 结构：**
```
Header: {"alg": "HS256", "typ": "JWT"}
Payload: {
  "sub": "username",
  "userId": 1,
  "username": "zhangsan",
  "iat": 1704067200,
  "exp": 1704153600
}
Signature: HMACSHA256(base64UrlEncode(header) + "." + base64UrlEncode(payload), secret)
```

**配置参数：**
- `jwt.secret`: 签名密钥（默认: mySecretKeyForJwtTokenMustBeLongEnough123456）
- `jwt.expire`: 过期时间（默认: 86400000ms = 24小时）

### 5.3 密码加密

使用 **BCryptPasswordEncoder** 进行密码哈希：
- 自动加盐
- 单向加密，不可解密
- 每次加密结果不同，但验证时可匹配

### 5.4 方法级权限控制

| 注解 | 作用 | 示例 |
|------|------|------|
| `@PreAuthorize("hasRole('ADMIN')")` | 仅管理员可访问 | 删除用户、创建图书 |
| `@PreAuthorize("isAuthenticated()")` | 已登录用户可访问 | 获取个人信息、借阅图书 |
| `@PreAuthorize("@userServiceImpl.isOwnUser(#id)")` | 仅本人可访问 | 获取个人详情 |

### 5.5 客户端 Token 管理

```
TokenStore (Bean)
    ├─ setToken(String token)    → 设置登录后的 Token
    ├─ getToken()                → 获取当前 Token
    ├─ clearToken()              → 清除 Token（退出登录时）
    └─ hasToken()                → 判断是否有 Token
```

### 6.4 数据库表结构

```sql
CREATE TABLE t_user (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) NOT NULL UNIQUE,
    password VARCHAR(100) NOT NULL,
    email VARCHAR(100),
    phone VARCHAR(11) NOT NULL UNIQUE,
    role VARCHAR(20) NOT NULL DEFAULT 'USER',
    create_time DATETIME
);

CREATE TABLE t_book (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    title VARCHAR(200) NOT NULL,
    author VARCHAR(100) NOT NULL,
    isbn VARCHAR(20) UNIQUE,
    publisher VARCHAR(100),
    publish_date DATETIME,
    category VARCHAR(50),
    description TEXT,
    stock INT NOT NULL,
    available INT NOT NULL,
    create_time DATETIME
);

CREATE TABLE t_borrow_record (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    book_id BIGINT NOT NULL,
    user_id BIGINT NOT NULL,
    borrow_time DATETIME NOT NULL,
    due_time DATETIME NOT NULL,
    return_time DATETIME,
    renew_count INT NOT NULL DEFAULT 0,
    status VARCHAR(20) NOT NULL DEFAULT 'BORROWED',
    FOREIGN KEY (book_id) REFERENCES t_book(id),
    FOREIGN KEY (user_id) REFERENCES t_user(id)
);
```

### 7.3 启动方式

#### 方式一：Maven 插件运行（推荐）

```bash
# 进入项目根目录
cd Library-System-JavaFX

# 使用 javafx-maven-plugin 启动
mvn javafx:run -pl client -am
```

> **说明**：`-pl client` 指定运行 client 模块，`-am` 会自动构建依赖的 common 模块。javafx-maven-plugin 会自动处理 module-path，确保 JavaFX 运行时正确加载。

#### 方式二：IDE 运行（需配置 VM 选项）

**IntelliJ IDEA 配置步骤：**

1. 打开运行配置（Run → Edit Configurations）
2. 找到 `FxApplication` 的运行配置
3. 在 `VM options` 中添加：

```
--module-path ${MAVEN_REPOSITORY}/org/openjfx/javafx-controls/21.0.2/javafx-controls-21.0.2.jar;${MAVEN_REPOSITORY}/org/openjfx/javafx-fxml/21.0.2/javafx-fxml-21.0.2.jar;${MAVEN_REPOSITORY}/org/openjfx/javafx-graphics/21.0.2/javafx-graphics-21.0.2.jar;${MAVEN_REPOSITORY}/org/openjfx/javafx-base/21.0.2/javafx-base-21.0.2.jar --add-modules javafx.controls,javafx.fxml,javafx.graphics
```

> 将 `${MAVEN_REPOSITORY}` 替换为你的实际 Maven 仓库路径（通常是 `~/.m2/repository`）

#### 方式三：Maven 运行配置（推荐 IDE 使用）

1. 在 IntelliJ IDEA 中，打开 `Maven` 面板（右侧边栏）
2. 展开 `nacon7-client` → `Plugins` → `javafx`
3. 右键点击 `javafx:run`，选择 `Run`

### 7.4 常见问题

#### 问题：缺少 JavaFX 运行时组件

**原因**：IDE 直接运行时未将 JavaFX 模块放到 module-path 上。

**解决方案**：使用 `mvn javafx:run` 命令启动，或在 IDE 运行配置中添加 VM 选项（见方式二）。

## 8. 开发规范

### 8.1 模块依赖规则

- **client** 模块包含所有业务逻辑（Repository、Service、Security）
- **common** 模块只能包含 entities、DTOs 和 utility classes

### 8.2 POM 规范

- 根 `pom.xml` 只包含 `dependencyManagement` 用于版本控制，不包含 `dependencies`
- **common** 模块需要 `jakarta.persistence-api` 依赖用于实体类
- **client** 模块需要 `spring-boot-starter-web`、`spring-boot-starter-data-jpa`、`spring-boot-starter-security` 和 JavaFX 依赖

### 8.3 新增功能开发规范

新增功能遵循以下实现顺序：
1. Entity（实体类）→ common 模块
2. DTO（数据传输对象）→ common 模块
3. Repository（数据访问层）→ client 模块
4. Service（业务逻辑接口）→ client 模块
5. ServiceImpl（业务逻辑实现）→ client 模块
6. View（界面组件）→ client 模块
